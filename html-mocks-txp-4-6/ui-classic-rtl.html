Rudimentary grid-type layouts—works best if inner content is of equal quantity (such as a grid of images). If you want to utilise this layout then there are some prerequisites:

  1. You must define a specific width for each .txp-grid-cell, either as a class attribute or style attribute, otherwise the cell width fits to the block content within it.
  2. The .txp-grid-cell should have a maximum width of no more than 294px (or its equivalent em measurement).
  3. As this layout uses inline-block, you will need to collapse the white space between each .txp-grid-cell to negate any unwanted margin between the cells.

Once CSS3 flexbox module is more settled and has improved browser support, this layout type can be made more robust.

Accessibility text new v4.6

<p class="txp-accessibility">...content...</p>

Accessibility text is hidden from the page but available to screen readers as an aid to those with visual impairments. This is accomplished via CSS visibility: hidden; not via display: hidden; (which would hide the text from most screen readers too).

Disabled non-form elements new v4.5

<p class="disabled" aria-disabled="true">...message...</p>

This is an example of an element greyed out via a .disabled class.

Please note that this class only gives a visual impression of disabled elements, any links and/or form elements therein would still be active and clickable. To truly disable form elements use the proper disabled attribute instead, see the forms section for more information.
